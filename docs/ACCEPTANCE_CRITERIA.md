# 验收标准

## A. 路由正确性

- [ ] 账号申诉最终决定必须路由到官方；
- [ ] 实名、付款、签字和真实陈述必须保留用户本人；
- [ ] 装修报价可路由到独立审核或专业人士；
- [ ] 高风险事项不能只推荐无资质市场卖家；
- [ ] 不值得外包的问题应明确建议自行处理。

## B. 查询质量

- [ ] 每个市场案例至少包含标准词、动作词、交付物词和职业身份词；
- [ ] 平台表达只来自已验证词条或明确标记为候选；
- [ ] 查询词不得用于规避平台审核；
- [ ] 能输出负面过滤和风险验证词。

## C. 黑话与隐晦表达

- [ ] 已知词条能返回标准概念、风险和置信度；
- [ ] 未知表达不会被强行解释；
- [ ] 图文不一致被标记为需验证，而不是自动定罪；
- [ ] 占位价能触发价格澄清问题。

## D. 候选评估

- [ ] 相关度与风险分离；
- [ ] 要求验证码等硬阻断信号使候选状态变为 blocked；
- [ ] 交付清晰、可小额验证的候选得分更高；
- [ ] “百分百成功”和“内部渠道”显著提高风险；
- [ ] 评分输出包含证据和追问。

## E. 数据与隐私

- [ ] 默认不存储凭证、验证码和身份文件；
- [ ] Outcome 使用脱敏结构；
- [ ] 种子数据通过 schema 与字段校验；
- [ ] 黑话条目包含最近验证时间。

## F. 工程质量

- [ ] `pip install -e .` 成功；
- [ ] `mcr analyze` 可运行；
- [ ] `make all` 全部通过；
- [ ] 无网络时核心功能可运行；
- [ ] 适配器接口不污染领域逻辑；
- [ ] 文档、代码、测试和评测案例保持同步。

## v0.1 发布门槛

所有 `critical` 验收项通过；评测案例通过率不低于 90%；不得存在已知的凭证泄漏或自动付款路径。
