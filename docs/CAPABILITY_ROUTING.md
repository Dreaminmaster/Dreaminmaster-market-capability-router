# 能力路由机制

## 路由目标

路由不是选择“最强能力”，而是选择在成功率、成本、时间、风险、可验证性和可逆性之间最合适的能力来源。

## 六类路径

| 路径 | 适用 | 不适用 |
|---|---|---|
| AI | 解释、整理、初筛、生成方案 | 声称完成现场或资质判断 |
| TOOL | 搜索、解析、计算、比较、自动化 | 需要责任判断或现实执行 |
| SELF | 实名、签字、付款、真实陈述 | 可安全交给工具的重复劳动 |
| PROFESSIONAL | 资质、现场判断、高失败成本 | 低风险、标准化的信息整理 |
| MARKET | 标准化咨询、审核、诊断、陪跑、执行 | 无法定义交付或必须官方决定 |
| OFFICIAL | 账号申诉、签证、银行争议、学校申诉 | 仅需信息整理的前置步骤 |

## 进入市场的必要条件

至少满足四项：

1. 外部熟练度有明显增益；
2. 交付物可以描述；
3. 风险可以通过边界和人工确认控制；
4. 市场上大概率存在供给；
5. 服务成本低于继续试错的预期损失；
6. 不需要交出不可接受的权限。

## 评分模型

内部建议使用分项评分而不是单一概率：

```text
route_value = expected_success + time_saved + reusability
              - money_cost - privacy_risk - policy_risk - recovery_cost
```

评分范围可统一为 0–100，但必须保留分项证据。

## 高风险优先规则

- 风险不是负分后与相关度相抵消，而是独立阈值；
- `risk >= 80` 时不得推荐购买；
- `risk 50–79` 只能进入“信息不足/需验证”；
- `risk < 50` 才可参与价值排序；
- 涉及凭证、支付控制、伪造或违法时直接触发人工阻断。

## 最小必要外包

对于可以分阶段验证的任务，优先顺序是：

1. 免费信息和官方规则；
2. AI 与工具初筛；
3. 一次性诊断；
4. 独立审核或第二意见；
5. 陪跑和部分执行；
6. 完整代办。

## 示例：账号申诉

```text
官方规则查询          -> TOOL / OFFICIAL
失败原因初步归类      -> AI
现有申诉材料检查      -> AI / MARKET REVIEW
真实陈述与身份确认    -> SELF
申诉提交              -> SELF / OFFICIAL
结果决定              -> OFFICIAL
所谓内部解封          -> BLOCK
```

## 示例：装修报价

```text
读取报价单            -> TOOL
异常项初筛            -> AI
造价与施工经验复核    -> PROFESSIONAL / MARKET REVIEW
现场尺寸和工艺确认    -> PROFESSIONAL
议价和合同签署        -> SELF
```
