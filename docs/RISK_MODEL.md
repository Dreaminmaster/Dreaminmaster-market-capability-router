# 风险与验证模型

## 风险类别

1. **权限风险**：密码、验证码、远程控制、支付权限；
2. **隐私风险**：身份证件、住址、合同、账号历史；
3. **资金风险**：平台外付款、虚假定金、不断加价；
4. **规则风险**：绕过实名、封禁、配额或平台政策；
5. **真实性风险**：伪造案例、模板评价、身份无法核实；
6. **交付风险**：交付物不明确、无限范围、结果不可验证；
7. **专业风险**：高风险事项由无资质人员处理；
8. **不可逆风险**：账号永久损失、错误提交、资金无法追回。

## 硬阻断信号

以下信号默认 `severity = critical`：

- 索取验证码或支付密码；
- 要求提供完整账号控制权；
- 伪造身份、材料、记录或证明；
- 声称可以修改平台后台结果；
- 明确要求绕过实名、风控或法律规则；
- 引导进行违法交易。

## 高风险信号

- 百分之百成功、包过、秒解；
- 所谓内部渠道但不解释交付；
- 低价占位后大幅加价；
- 要求脱离平台担保；
- 拒绝说明交付物；
- 案例无法脱敏验证；
- 只制造紧迫感和恐惧。

## 风险输出

每个风险项包含：

```text
rule_id
category
severity
matched_evidence
explanation
recommended_action
human_confirmation_required
```

## 数据最小化

默认只保存：

- 脱敏后的问题类型；
- 路由与查询词；
- 风险标签；
- 结果统计。

默认不保存：

- 密码、验证码；
- 身份证号、银行卡号；
- 完整聊天记录；
- 可识别卖家身份；
- 未经授权的个人资料。

## 验证问题模板

- 你提供的是诊断、建议、审核还是代操作？
- 最终会交付什么文件或结论？
- 是否需要账号密码、验证码、身份证件或远程控制？
- 当前价格是总价、定金还是咨询费？
- 无法完成时如何收费？
- 是否可以先用脱敏资料进行初步判断？
- 是否可以在平台担保内完成交易？
