# 产品规格：Market Capability Router

## 1. 产品定义

Market Capability Router（MCR）是一套“现实能力发现、采购、验证和调度系统”。当用户面对复杂事务时，系统先识别真实目标和摩擦，再判断每个子任务应由 AI、工具、用户本人、专业人士、市场服务或官方机构承担。

系统的核心不是让用户购买更多服务，而是降低错误自学、错误外包、错误搜索和错误信任造成的认知学费。

## 2. 核心问题

普通搜索有一个前提：用户已经知道该搜索什么。但现实中常见的问题是：

- 不知道某类服务存在；
- 不知道服务的正式名称；
- 不知道拥有这种能力的人是什么职业；
- 不知道平台上的实际表达、简拼和隐晦说法；
- 不知道应该买诊断、审核、陪跑还是完整代办；
- 不知道哪个候选有效、哪个只是占位、引流或诈骗；
- 不知道官方渠道其实更合适。

## 3. 目标用户

- 面对陌生生活事务的普通用户；
- 需要快速判断是否外包的个人或小团队；
- 使用本地 Agent、OpenClaw、Claude Code、Codex 等系统的用户；
- 需要将线上信息能力连接到现实执行能力的智能助手开发者。

## 4. 典型场景

1. 装修报价审核与增项风险；
2. 平台账号申诉材料诊断；
3. 酒店预订合法渠道比较；
4. 二手电子产品验机；
5. 搬家、看房、现场拍摄等异地执行；
6. 合同、简历、申请材料的第二意见；
7. 软件安装、开源项目调试与远程排查。

## 5. 主要输出

每次分析必须产生：

- 真实目标；
- 摩擦类型和证据；
- 子任务列表；
- 每个子任务的能力路由；
- 是否值得进入市场；
- 推荐的服务层级；
- 查询网格；
- 候选验证问题；
- 风险与人工确认点；
- 可选的结果记录。

## 6. 服务层级

系统优先使用“最小必要外包”：

```text
AI 初筛 -> 工具验证 -> 低价诊断 -> 独立审核/第二意见
-> 陪跑或部分执行 -> 完整代办
```

除非任务本身只能由专业人员完成，否则不得直接默认完整代办。

## 7. 非目标

第一版不负责：

- 自动登录闲鱼或其他平台；
- 自动付款、签字或提交身份材料；
- 自动与卖家进行无限轮私聊；
- 规避平台审核；
- 寻找违法、欺诈或侵权服务；
- 将不确定黑话直接标记为事实；
- 替代医生、律师、注册会计师等专业判断。

## 8. 成功指标

产品指标不应只看搜索点击率，而应看：

- 用户是否发现此前不知道的有效路径；
- 是否减少高价完整代办，转为更低风险的诊断或审核；
- 查询词是否提高有效候选比例；
- 高风险候选是否被正确标记；
- 最终交付是否符合承诺；
- 实际花费、时间和失败次数是否下降；
- 用户能否复用所获得的知识，而非形成依赖。

## 9. 第一版范围

v0.1 提供本地规则引擎和结构化数据：

- 六类摩擦诊断；
- 六类能力路由；
- 查询网格；
- 平台语言词条识别；
- 风险规则；
- 候选评分；
- JSONL 结果存储；
- Skill 和 CLI；
- 评测案例。

模型、搜索、浏览器、OCR、图像理解作为后续适配器接入。
