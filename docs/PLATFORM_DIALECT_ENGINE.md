# Platform Dialect Engine

## 目标

平台语言引擎解决“用户语言、行业语言和平台卖家语言不一致”的问题。它负责理解和筛选，不负责帮助规避审核。

## 可识别表达

- 拼音首字母和简拼；
- 谐音、同音替代、错别字；
- 拆字、符号和 emoji；
- 职业身份暗示；
- 图片中真实服务、标题中无关商品；
- 占位价、定金价、咨询价和总价不一致；
- “懂的来”“具体私聊”“拍前联系”等不完整表达；
- 评论和聊天中出现的真实交付。

## 词条结构

```json
{
  "term": "报价体检",
  "canonical_concept": "装修报价审核",
  "platform": "generic_marketplace",
  "expression_type": "packaged_phrase",
  "possible_services": ["报价复核", "增项检查"],
  "risk_level": "low",
  "confidence": 0.88,
  "first_seen": "2026-06-14",
  "last_verified": "2026-06-14",
  "evidence_required": ["description", "deliverable"]
}
```

## 推断流程

1. 词典匹配：识别已知表达；
2. 组合信号：标题、描述、图片、价格、卖家历史、评价和聊天；
3. 多假设输出：保留多个可能含义；
4. 置信度和证据：说明为什么这样理解；
5. 人工确认：低置信度不得写回正式词库；
6. 时效衰减：长期未验证的词条降低权重。

## 图文不一致

图文不一致不是自动判定违规，而是提高“需要验证”分。系统应追问：

- 标题商品与实际交付有什么关系；
- 当前标价是总价、定金还是咨询费；
- 是否为虚拟服务；
- 交付在哪里完成；
- 是否需要离开平台交易。

## 查询网格

查询词来自六个维度：

```text
问题 × 动作 × 交付物 × 人员身份 × 平台表达 × 风险过滤
```

例如装修报价：

```text
问题：装修报价、装修增项
动作：审核、复核、排雷、把关
交付：标注报告、风险清单、追问清单
人员：造价员、监理、老工长
平台词：看单、体检、过一遍
过滤：包过、内部、先平台外付款
```

## 安全约束

系统可以识别“用曲别针包装其他服务”的现象并说明风险，但不得：

- 生成用于发布规避审核商品的暗语；
- 教用户如何逃避平台检测；
- 帮助寻找违法、侵权或欺诈服务；
- 将“内部渠道”包装成正常能力；
- 根据单一暗语对个人作确定性指控。
